# PriceBuddy 비즈니스 모델 및 재설계

## 🎯 핵심 가치 제안 (Value Proposition)

**PriceBuddy는 글로벌 쇼핑몰 가격 비교 플랫폼입니다.**

### 사용자 관점
- **한 곳에서 모든 쇼핑몰 가격 비교**: 쿠팡, 네이버, 아마존, 이베이 등 전 세계 쇼핑몰 가격을 실시간으로 비교
- **최저가 자동 알림**: 원하는 가격에 도달하면 자동으로 알림
- **가격 변동 추적**: 상품 가격의 과거/현재/예측 가격을 한눈에 확인
- **캐시백 자동 적립**: 제휴 링크를 통해 구매 시 자동으로 캐시백 적립

### 비즈니스 관점
- **제휴 수수료 (Affiliate Commission)**: 주요 수익원
- **프리미엄 구독**: 고급 기능 (무제한 알림, 가격 예측, 우선 지원)
- **광고 수익**: 관련 상품 추천 및 배너 광고

## 💰 수익 모델

### 1. 제휴 수수료 (Affiliate Commission) - **주요 수익원 (70%)**

각 마켓플레이스에서 구매 시 발생하는 수수료:

| 마켓플레이스 | 평균 수수료율 | 월 예상 거래액 | 예상 수익 |
|------------|------------|--------------|----------|
| 쿠팡 파트너스 | 5-10% | 1억원 | 500만원 |
| 네이버 쇼핑 | 3-7% | 5천만원 | 200만원 |
| 아마존 어소시에이트 | 4-8% | 3천만원 | 150만원 |
| 이베이 파트너 네트워크 | 3-5% | 2천만원 | 80만원 |
| **합계** | | **2억원** | **930만원/월** |

**사용자에게는 캐시백으로 50% 지급, 플랫폼은 50% 수익**

### 2. 프리미엄 구독 (20%)

| 플랜 | 가격 | 기능 | 예상 가입률 |
|------|------|------|-----------|
| Basic | 무료 | 기본 검색, 5개 알림 | - |
| Pro | 9,900원/월 | 무제한 알림, 가격 예측, 광고 제거 | 5% |
| Pro Plus | 19,900원/월 | Pro + 우선 지원, API 접근 | 1% |

**10,000명 사용자 기준:**
- Pro: 500명 × 9,900원 = 495만원/월
- Pro Plus: 100명 × 19,900원 = 199만원/월
- **합계: 694만원/월**

### 3. 광고 수익 (10%)

- 관련 상품 추천 (네이티브 광고)
- 배너 광고
- **예상: 100만원/월**

## 🏗️ 재설계된 아키텍처

### 핵심 원칙
1. **실제 작동하는 검색 우선**: 사용자가 바로 사용할 수 있어야 함
2. **수익 모델 중심**: 제휴 링크가 핵심이므로 모든 링크는 제휴 링크로 변환
3. **단계적 확장**: 초기에는 한국 시장(쿠팡/네이버) 집중, 이후 글로벌 확장

### 시스템 구조

```
┌─────────────────────────────────────────────────────────┐
│                    Frontend (Web App)                    │
│  - React 19 + Vite                                      │
│  - 검색, 상품 상세, 가격 비교, 알림 설정                │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│              Firebase Functions (API Gateway)           │
│  - /search: 검색 요청 라우팅                            │
│  - /getProduct: 상품 상세 조회                          │
│  - /priceTracking: 가격 추적                            │
│  - /alerts: 알림 관리                                   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│              Scraper Service (Cloud Run)                │
│  - 쿠팡/네이버 실시간 검색                              │
│  - 상품 페이지 스크래핑                                 │
│  - 가격 정보 추출                                        │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    Firestore (DB)                       │
│  - products: 상품 정보                                  │
│  - offers: 각 마켓플레이스별 가격                       │
│  - price_history: 가격 변동 히스토리                    │
│  - alerts: 사용자 알림 설정                             │
└─────────────────────────────────────────────────────────┘
```

## 🚀 단계별 개발 계획

### Phase 1: 핵심 기능 구현 (1-2주) ⚡ **우선순위 최고**

**목표**: 사용자가 실제로 검색하고 가격을 비교할 수 있게 만들기

1. **실제 마켓플레이스 검색 구현**
   - 스크래퍼 서비스 배포 (Cloud Run)
   - 쿠팡/네이버 실시간 검색 연동
   - 검색 결과를 Firestore에 저장

2. **제휴 링크 변환**
   - 모든 상품 링크를 제휴 링크로 자동 변환
   - 쿠팡 파트너스 API 연동
   - 네이버 쇼핑 API 연동

3. **가격 비교 UI 개선**
   - 실시간 가격 표시
   - 배송비 포함 총액 계산
   - 최저가 강조 표시

### Phase 2: 사용자 경험 개선 (1주)

1. **가격 알림 시스템**
   - 목표 가격 설정
   - 가격 하락 시 자동 알림 (이메일/푸시)
   - 알림 히스토리 관리

2. **가격 추적**
   - 가격 변동 그래프
   - 가격 예측 (간단한 ML 모델)

### Phase 3: 수익화 (1주)

1. **프리미엄 구독**
   - 결제 시스템 연동 (포트원)
   - 구독 관리
   - 기능 제한 해제

2. **캐시백 시스템**
   - 구매 확인 (제휴 링크 추적)
   - 캐시백 적립
   - 출금 기능

### Phase 4: 확장 (2주)

1. **글로벌 마켓플레이스 추가**
   - 아마존 (미국, 일본)
   - 이베이
   - 라쿠텐

2. **고급 기능**
   - AI 가격 예측
   - 리뷰 분석
   - 상품 추천

## 🔧 즉시 수정해야 할 사항

### 1. 검색 기능 완전히 재구현

**현재 문제:**
- Firestore에만 의존 (실제 마켓플레이스 검색 안함)
- 스크래퍼 서비스가 배포되지 않음
- 검색 결과가 없음

**해결 방법:**
1. 스크래퍼 서비스를 Cloud Run에 배포
2. Firebase Functions의 `/search` 엔드포인트를 스크래퍼 서비스와 연동
3. 검색 결과를 실시간으로 가져와서 표시

### 2. 제휴 링크 시스템 구축

**현재 문제:**
- 제휴 링크 변환이 없음
- 수익 모델이 작동하지 않음

**해결 방법:**
1. 쿠팡 파트너스 API 연동
2. 네이버 쇼핑 API 연동
3. 모든 상품 링크를 제휴 링크로 자동 변환

### 3. 데이터 품질 개선

**현재 문제:**
- 이미지 URL이 잘못됨 (404 에러)
- 가격 정보가 부정확함
- 상품 정보가 불완전함

**해결 방법:**
1. 스크래퍼에서 실제 이미지 URL 추출
2. 가격 정보 실시간 업데이트
3. 상품 정보 검증 로직 추가

- **DAU (Daily Active Users)**: 일일 활성 사용자 수

- **프리미엄 구독률**: 전체 사용자 대비 구독자 비율
